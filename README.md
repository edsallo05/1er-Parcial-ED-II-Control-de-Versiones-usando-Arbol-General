# 🌳 Sistema de Control de Versiones con Árboles

## 📌 Descripción
Este proyecto implementa una estructura de datos tipo árbol general en Java para simular un sistema de control de versiones.  
Cada nodo del árbol representa una versión de un documento, permitiendo la creación de múltiples ramas a partir de una misma versión.

## 🎯 Objetivo
Aplicar el uso de árboles para resolver una problemática real, demostrando cómo esta estructura permite modelar relaciones jerárquicas y ramificadas de manera eficiente.

## 🧠 Estructura de datos utilizada
Se utilizó un *árbol general (n-ario)* donde:
- Cada nodo representa una versión
- Cada nodo puede tener múltiples hijos
- Existe una raíz que representa la versión inicial

## ⚙️ Funcionalidades principales
- Crear versión inicial
- Agregar nuevas versiones
- Cambiar versión actual
- Buscar versiones por ID (DFS recursivo)
- Mostrar el árbol de versiones en consola

## 🔍 Conceptos aplicados
- Recursividad
- Recorrido en profundidad (DFS)
- Estructuras jerárquicas

## 🚀 Conclusión
El uso de árboles permite representar de forma natural el historial de versiones con ramificaciones, facilitando la navegación y organización de la información.

## 🛠️ Tecnologías
- Java
- Git
